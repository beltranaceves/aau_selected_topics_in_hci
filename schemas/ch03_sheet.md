# I/O on Mobile Devices

    TOUCH
      Problems: no haptics, occlusion, low precision, no hover, reachability
      Solutions:
        - Haptics: Tactus fluid overlay; electrovibration
        - Occlusion: hand model; back-of-device input
        - Precision: PIP model; offset cursor; Shift; data-driven maps; prediction keyboards
        - Hover: lift-off strategy; pre-touch sensing
        - Reachability: OS one-handed modes

    MOTION INPUT
      IMU (accel, magnetometer, gyro) + sensor fusion
      MEMS principles; calibration; EMG; free-air gestures; deformation; bumping

    MOTION OUTPUT
      Vibration; multi-motor cues; GVS; muscle-force feedback; shape-changing devices

    VISUAL OUTPUT
      LCD (needs backlight), AMOLED (self-lit), E-Ink (bistable, slow)

    VISUAL INPUT
      Camera hardware; barcodes; OCR; SLAM; CNN object detection; touch projector

    OTHER CHANNELS
      Biosensing; body stance; grasp; facial expression; gaze; heart rate; audio FFT
